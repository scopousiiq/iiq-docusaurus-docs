Creates a new category. The CategoryTypeId in the request body determines what type of category is created (Asset, Issue, Model, Kb, etc.). Use the type-specific endpoints like [POST /api/v1.0/categories/for/issue/new](/docs/api/categories/create-issue-category) when you want the server to assign the CategoryTypeId automatically.

**Prerequisites**
1. **CategoryTypeId** - Use [POST /api/v1.0/categories/v2](/docs/api/categories/search-categories-v-2) to list existing categories and capture a valid `Items[].CategoryTypeId`.
2. **ParentCategoryId** (optional) - To create a child category, extract a parent's `Items[].CategoryId` from the search.

**Workflow Example**
1. Discover category types from existing categories.
2. [POST /api/v1.0/categories/new](/docs/api/categories/create-category) with Name, CategoryTypeId, and optional ParentCategoryId.

**Minimal Required Fields**: Name, CategoryTypeId.
