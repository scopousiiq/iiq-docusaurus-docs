Performs a targeted search across three core entities—Tickets, Assets, and Users—using a single query string. This endpoint is designed for 'scanning' scenarios where a quick lookup of an identifier (like a ticket number, serial number, or school ID) is required across multiple domains.

**Search Behavior**
- **Tickets**: Searches specifically for a ticket by its `TicketNumber` matching the query.
- **Assets**: Searches for assets by `SerialNumber`, `AssetTag`, or `Name` matching the query.
- **Users**: Searches for users by `SchoolIdNumber` matching the query.

**Response Structure**
Returns an `ItemGetResponse` containing a `SearchResults` object with separate lists for `Tickets`, `Assets`, and `Users`.
