Bitmask representing which entity types to include in the global search. Sum the values of desired entities to search multiple types simultaneously.

**Bitmask Values:**
| Entity Type | Bit Value | Description |
|-------------|-----------|-------------|
| None | 0 | No entities |
| Tickets | 1 | Help desk tickets |
| KbArticles | 2 | Knowledge base articles |
| Users | 4 | User accounts |
| Assets | 8 | Hardware/software assets |
| Locations | 16 | Schools, buildings, rooms |
| RepairOrderNumber | 32 | Repair order tracking numbers |
| Students | 64 | Student records |
| Models | 128 | Device/asset models |
| AssetFundingTypes | 256 | Asset funding classifications |
| AssetStatusTypes | 512 | Asset status types |
| AssetTypes | 1024 | Asset type classifications |

**Examples:**
- Search Tickets only: `1`
- Search Tickets and Assets: `9` (1 + 8)
- Search Tickets, KB Articles, and Assets (default): `11` (1 + 2 + 8)
- Search Users and Students: `68` (4 + 64)
- Search all common entities: `31` (1 + 2 + 4 + 8 + 16)
