Creates a new site-scoped view definition used by site grids. Use this to persist a reusable layout with filters, columns, and sort order for site views.

**Prerequisites**
1. **SiteId/ProductId** - Use [GET /api/v1.0/sites/\{siteUrl\}](/docs/api/sites/get-site-by-url) and extract `Item.SiteId` plus a `ProductId` from `Item.LicensedProducts[]`.
2. **UserId** - Use [POST /api/v1.0/search](/docs/api/users/search-users) to find the owner and extract `Item.Users[].UserId`.
3. **ViewTypeId** - Use [GET /api/v1.0/views/for-sites](/docs/api/views/list-site-views) to copy a valid `Items[].ViewTypeId` for site views.

**Workflow Example**
1. Bootstrap the site: [GET /api/v1.0/sites/\{siteUrl\}](/docs/api/sites/get-site-by-url) -&gt; capture SiteId and ProductId.
2. (Optional) Inspect existing site views: [GET /api/v1.0/views/for-sites](/docs/api/views/list-site-views).
3. Create the view: [POST /api/v1.0/views/for-sites/new](/docs/api/views/create-site-view) with a ViewDefinition payload.

**Minimal Required Fields**: `ViewId`, `IsDeleted`, `SiteId`, `ProductId`, `UserId`, `Name`, `ViewTypeId`, `PageSize`.
