Creates a new saved view definition for any supported view type when using the generic views endpoint. Use this for programmatic creation across tickets, assets, users, or site contexts.

**Prerequisites**
1. **SiteId/ProductId** - Use [GET /api/v1.0/sites/\{siteUrl\}](/docs/api/sites/get-site-by-url) to obtain `Item.SiteId` and a `ProductId` from `Item.LicensedProducts[]`.
2. **UserId** - Use [POST /api/v1.0/search](/docs/api/users/search-users) to identify the owner and extract `Item.Users[].UserId`.
3. **ViewTypeId** - Copy a valid type from [GET /api/v1.0/tickets/views](/docs/api/tickets/list-ticket-views), [GET /api/v1.0/assets/views](/docs/api/assets/list-asset-views), [GET /api/v1.0/users/views](/docs/api/users/list-user-views), or [GET /api/v1.0/views/for-sites](/docs/api/views/list-site-views).

**Workflow Example**
1. Bootstrap site context: [GET /api/v1.0/sites/\{siteUrl\}](/docs/api/sites/get-site-by-url) -&gt; capture SiteId and ProductId.
2. Inspect existing views to choose a ViewTypeId.
3. [POST /api/v1.0/views/new](/docs/api/views/create-view) with a ViewDefinition payload.

**Minimal Required Fields**: `ViewId`, `IsDeleted`, `SiteId`, `ProductId`, `UserId`, `Name`, `ViewTypeId`, `PageSize`.
