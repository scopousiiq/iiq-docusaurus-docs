Updates an existing site view definition. The payload replaces filters, columns, sorting, and metadata for the specified viewId.

**Prerequisites**
1. **viewId** - Use [GET /api/v1.0/views/for-sites](/docs/api/views/list-site-views) to list site views and extract `Items[].ViewId`.

**Workflow Example**
1. Fetch the current definition: [GET /api/v1.0/views/for-sites/\{viewId\}](/docs/api/views/get-site-view).
2. Modify the ViewDefinition (Name, Filters, Columns, Sort, PageSize).
3. Save the changes: [POST /api/v1.0/views/for-sites/\{viewId\}](/docs/api/views/update-site-view).

**Minimal Required Fields**: `viewId` (path), plus ViewDefinition fields `ViewId`, `IsDeleted`, `SiteId`, `ProductId`, `UserId`, `Name`, `ViewTypeId`, `PageSize`.
