Universal filter structure used across all search endpoints (Tickets, Assets, Users). Each FilterMatch targets a specific facet and provides criteria through either the `Id` field (for entity references) or the `Value` field (for expressions).

## Supported Entity Types

| Entity | Facet Key Reference | Total Facets |
|--------|---------------------|-------------|
| Tickets | `TicketSearchFacetKey` | 103 |
| Assets | `AssetSearchFacetKey` | 75 |
| Users | `UserSearchFacetKey` | 41 |

## Field Usage by Facet Type

| Facet Type | Use Field | Example |
|------------|-----------|--------|
| Entity reference (status, agent, location) | `Id` | `{"Facet": "status", "Id": "guid-here"}` |
| Date expression | `Value` | `{"Facet": "createddate", "Value": "range:thisweek"}` |
| Numeric expression | `Value` | `{"Facet": "purchasedprice", "Value": "numoperator:greaterthan:100"}` |
| Keyword search | `Value` | `{"Facet": "keyword", "Value": "laptop"}` |
| Boolean facet | `Value` | `{"Facet": "isurgent", "Value": "true"}` |

## Filter Combination Logic

- **Default (AND)**: All filters are combined with AND logic
- **OR within groups**: Filters with the same `GroupIndex` are combined with OR logic
- **Exclusion**: Set `Negative: true` to exclude matching records

## Expression Syntax Reference

See `DateExpressionSyntax` and `NumericExpressionSyntax` schemas for detailed syntax documentation.

## Universal Facets

Six facets are shared across all search endpoints: `grade`, `location`, `locationtype`, `role`, `source`, `user`. See `UniversalFacets` schema for detailed cross-entity behavior documentation.
